Die Idee hinter iMeVS

Vorgeschichte

Ein kurzer Abstecher in die Geschichte, wie mir die Idee zu iMeVS kam: Ich laufe gemütlich ohne mir Böses zu denken die Treppe an einer Bahnhaltestelle nach unten. War abgelenkt und habe nicht auf den Boden geschaut. Vandalen haben eine Reihe von Steinen direkt vor der Treppe entfernt und ich trete natürlich rein. Hätte ich das Treppengeländer nicht zu fassen gekriegt, hätte ich mich wohl ganz schön verletzt. Die Suche nach einer Möglichkeit zum Melden brauchte über eine halbe Stunde - Zeit, die kaum einer investiert, um Schäden und Störungen im öffentlichen Raum zu melden.

Was soll iMeVS werden?

iMeVS soll aus zwei Teilen bestehen: Einer für jeden nutzbaren Programmierschnittstelle und aus einer App für Android, Blackberry, Windows Phone, Firefox OS und Ubuntu Touch. Eine iOS-App von uns ist aus Gründen der Finanzierung - keine Werbung, keine anderen Einnahmen durch die App - vermutlich nicht möglich.

Die API soll anhang der Postleitzahl einer GPS-Position die zuständige Stelle für das Problem finden und diese per E-Mail informieren, die App soll die Daten sammeln und an die API senden.

Wie soll iMeVS umgesetzt werden?

Backend (Schnittstelle)

Die API bzw. das gesamte Backend soll auf Basis von PHP ohne ein Framework entwickelt werden. Die zuständigen Stellen werden in einer nicht-öffentlichen Datenbank gespeichert. Die Antwort der API soll im JSON-Format erfolgen.

App

Die iMeVS-App soll mit Phonegap und Phonegap-Build für alle gängigen Plattformen, außer iOS, entwickelt und veröffentlicht werden. Aus Geldgründen wird eine iOS-App vermutlicht nicht angeboten werden - 99 USD pro Jahr für einen Developer-Account sind eine Menge, wenn man mit der App keinerlei Einnahmen macht. Wenn wir aber einen Publisher, der uns die App veröffentlicht, oder einen Sponsor, der uns das Geld zur Verfügung stellt, finden, werden wir die App auch für iOS anbieten.
